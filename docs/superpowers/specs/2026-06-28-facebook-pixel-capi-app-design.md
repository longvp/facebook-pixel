# Facebook Pixel & CAPI Manager — Shopify Embedded App (Design)

- **Date**: 2026-06-28
- **Status**: Approved (design), pending spec review
- **Stack**: Node + Remix + Polaris React (`@shopify/polaris`) — Shopify embedded app

## 1. Purpose

A Shopify embedded app that lets a merchant manage Facebook Pixels and the
Conversions API (CAPI) from inside the Shopify admin. Merchant can:

- CRUD pixels (Facebook Pixel ID + friendly name)
- Enable CAPI per pixel with an access token (stored encrypted)
- Choose which pages are tracked: **All / Selected / Excluded**
- Toggle each pixel **active / inactive**

Browser-side events fire through a Web Pixel Extension; server-side events fire
through the Conversions API driven by Shopify webhooks. The two are deduplicated
via a shared `event_id`.

## 2. Architecture Decisions

| Area | Decision |
|------|----------|
| Persistence | Standalone — Prisma ORM |
| Database | MySQL (managed locally via MySQL Workbench) |
| Tracking | Web Pixel Extension (browser) + server-side CAPI (webhooks) |
| Dev flow | Slash commands: `/brainstorm → /plan → /userstories → /classify → /execute → /verify` |
| Scaffold | Official Shopify CLI (`shopify app init`, Remix template) |

The app does **not** depend on the existing Go backend at
`d:/Xipat/facebook-shopify-backend`; it is self-contained.

## 4. Data Model (Prisma → MySQL)

```prisma
// Shopify session storage (from template)
model Session { /* shop, accessToken, scope, etc. */ }

enum TrackingMode { ALL  SELECTED  EXCLUDED }

model Pixel {
  id            String        @id @default(cuid())
  shop          String        @db.VarChar(255)
  name          String
  pixelId       String        // Facebook Pixel ID
  capiEnabled   Boolean       @default(false)
  accessToken   String?       @db.Text   // CAPI token, encrypted at rest
  testEventCode String?
  trackingMode  TrackingMode  @default(ALL)
  trackingPages Json          // page paths/handles for SELECTED / EXCLUDED
  active        Boolean       @default(true)
  createdAt     DateTime      @default(now())
  updatedAt     DateTime      @updatedAt

  @@index([shop])
}
```

**Security**: `accessToken` is encrypted before persistence (AES via an app
secret in env) and never returned to the client in plaintext.

## 5. Remix App

- `app._index` — pixel list (Polaris `IndexTable`) with active/inactive toggle
- `app.pixels.new` / `app.pixels.$id` — CRUD form: Pixel ID, name, CAPI toggle +
  token, test event code, tracking mode + page selector
- `app/services/capi.server.ts` — send events to the Facebook Conversions API
  (SHA-256 hash PII, attach `event_id` for browser/server dedup)
- `app/services/pixel.server.ts` — CRUD + token encrypt/decrypt
- `app/routes/webhooks.*` — `orders/create` → Purchase event; `app/uninstalled` → cleanup
- `extensions/web-pixel/` — sandbox: subscribe to browser events, fire the
  browser pixel honoring `trackingMode` + `active`, attach `event_id`

## 5a. UI / UX (reference: `docs/ui/pixel-app-ui.html`)

The approved visual design is the static mockup at `docs/ui/pixel-app-ui.html`.
Rebuild it with **Polaris React** (`@shopify/polaris`) — the official React
component library — wrapped in App Bridge (`@shopify/app-bridge-react`). The
mockup hand-rolls Polaris design tokens for illustration only; the real app must
use actual Polaris React components (`Page`, `Card`, `IndexTable`, `FormLayout`,
`TextField`, `ChoiceList`, `Checkbox`, `Badge`, `Banner`, `Button`, `Frame` +
`Toast`, `Modal`), not hand-written CSS. Import `@shopify/polaris/build/esm/styles.css`
and wrap the app in `<AppProvider>`. Match layout, copy, and behavior of the
mockup. Two screens, single-app surface:

### List screen (`app._index`)
- Page header: app logo mark + "Facebook Pixel & CAPI" title.
- Card toolbar: search input (placeholder "Search by pixel name, pixel ID") +
  primary **"Add pixel"** button.
- Table columns: **Active** (toggle) · **Pixel ID** (monospace) · **Pixel name** ·
  **Pages** (badge: All / Selected / Excluded pages) · **Conversion API** (toggle) ·
  **Actions** (edit + delete icon buttons).
- Pagination: page-size select (5 / 10 / 20), prev/next, `current/total` info.
- Footer note with knowledge-base / support links.
- Empty state: "No pixels found" row.

### Add / Edit screen (`app.pixels.new` / `app.pixels.$id`)
- Breadcrumb back to list; title "Add Facebook pixel" / "Edit Facebook pixel".
- **Details card**:
  - Pixel name — required, max 255, char counter.
  - Pixel ID — required, max 20, char counter, "How I get it?" help link.
    **Immutable on edit** (field disabled when editing).
  - Tracking on pages — bordered radio group: **All pages / Selected page /
    Excluded page**.
- **CAPI card** (collapsible):
  - Header: Facebook icon, "Conversions API (solution for iOS 14.5)" +
    "Recommended" badge, descriptive subtitle, enable toggle.
  - Body (revealed when toggle on): Facebook access token (password input, hint
    re: System User Token + `ads_management`/`ads_read`), Test event code
    (max 20, char counter) with a warning banner to remove it after testing.
- Form actions (sticky bar): **Discard** / **Save pixel**.

### Behavior rules
- Enabling CAPI (toggle on list **or** form) requires a saved access token first;
  otherwise show a warning toast and revert the toggle.
- Toggling Active / CAPI, save, and delete each surface a **toast**.
- Delete opens a **confirmation modal** ("This action cannot be undone").
- Pixel ID is unique per shop; duplicate on create → warning.
- Validation: name + Pixel ID required before save.

## 6. `.claude` Tooling

- **CLAUDE.md** — stack, structure, common commands (`npm run dev`,
  `prisma migrate dev`, `shopify app dev`), code conventions, the 6-step flow.
- **settings.json hooks**:
  - `PostToolUse` on Edit/Write of `*.ts|*.tsx` → run Prettier
  - `permissions.allow`: `npm`, `npx`, `shopify`, `prisma`, `node`
- **commands** — each defines goal + checklist + output path under `docs/flow/`.
  `/classify` groups tasks into `now / complex / discuss` and writes
  `docs/flow/complex-cases.md` and `docs/flow/questions.md`.
- **skills** — `facebook-capi` (event schema, hashing, dedup) and
  `shopify-remix-patterns`.

## 7. Build Sequence

1. Create `.claude` tooling + `CLAUDE.md` + `docs/flow/` (no network needed).
2. **Scaffold** with Shopify CLI — *interactive* (pick Remix template, Partner
   login via browser). Requires user participation.
3. Switch Prisma datasource to MySQL, define `Pixel`, run `prisma migrate dev`.
4. Implement features (CRUD / CAPI / extension) later via the `/execute` flow.

## 7a. Open Items (surface into `docs/flow/questions.md` during /classify)
